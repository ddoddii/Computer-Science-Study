### **1. 쿠키와 세션의 차이에 대해 설명해 주세요.**
쿠키는 클라이언트에 저장되고 세션을 서버에 저장됩니다. 또한 쿠키는 도메인 단위로 적용되기에 한 도메인에서 생성된 쿠키는 해당 도메인의 모든 페이지에 접근 할 수 있습니다. 세션은 사용자별로 별도의 세션을 유지하며 세션 ID를 통해 사용자를 식별합니다.
- **세션 방식의 로그인 과정에 대해 설명해 주세요.**
  - 사용자가 로그인에 성공하면 서버는 사용자의 로그인 상태를 기억하기 위해 고유한 세션 ID를 생성합니다. 서버는 세션 ID를 클라이언트에게 전달하고 이는 쿠키를 통해 클라이언트에 저장됩니다. 클라이언트는 이후 요청에서 세션 ID를 서버에 전달합니다.
- **HTTP의 특성인 Stateless에 대해 설명해 주세요.**
  - 서버가 클라이언트의 상태를 기억하지 않아 클라이언트와 서버 간의 통신에서 각각의 요청이 독립적으로 처리되는 특성을 뜻합니다. 이전 요청에 대한 정보를 서보가 유지 하지 않습니다. 
- **Stateless의 의미를 살펴보면, 세션은 적절하지 않은 인증 방법 아닌가요?**
  - 세션을 사용하기 위해서는 클라이언트가 세션 ID를 각각의 요청에서 전달해야하기에 절절하지 않은 인증 방법으로 여겨질 수 있습니다. 이를 위한 대한으로 토큰 기반 인증이 많이 사용되기도 합니다.
- **규모가 커져 서버가 여러 개가 된다면, 세션을 어떻게 관리할 수 있을까요?**
  - 여러 서버 간에 세션 정보를 공유하기 위해 세션 클러스터링과 같은 방법을 사용할 수 잇습니다.
- **토큰 기반 방식은 어떤 장점이 있을까요?**
  - 보안성 측면에서 큰 장점을 가집니다. 중요한 정보를 토콘을 사용하기 때문에 보안상의 위험을 줄일 수 있습니다. 또한 서버는 사용자의 세션 정보를 저장하거나 관리할 필요가 없으므로 서버의 부담도 줄어드는 장점이 있습니다. 

### **2. HTTP 응답코드에 대해 설명해 주세요.**
웹 서버가 클라이언트의 요청에 대한 처리 결과를 전달하는 데 사용되는 코드입니다.  클라이언트는 이 응답 코드를 통해 요청이 성공적으로 처리되었는지, 오류가 발생했는지 등을 파악할 수 있습니다.
- **401 (Unauthorized) 와 403 (Forbidden)은 의미적으로 어떤 차이가 있나요?**
  - 401(Unauthorized)은 클라이언트가 요청한 리소스에 접근하기 위해서는 인증이 필요하다는 것을 의미합니다. 403(Forbidden)은 클라이언트가 요청한 리소스에 대한 접근 권한이 서버에 의해 거부되었음을 나타냅니다. 즉, 클라이언트가 인증에 성공했더라도 서버가 특정 조건을 충족하지 않는다고 판단하여 접근을 거부한 경우에 사용됩니다.
- **200 (ok) 와 201 (created) 의 차이에 대해 설명해 주세요.**
  - 200 (OK)는 클라이언트의 요청이 성공적으로 처리되었음을 의미합니다.201 (Created)은 클라이언트의 요청을 통해 새로운 리소스가 성공적으로 생성되었음을 나타냅니다.
- **필요하다면 저희가 직접 응답코드를 정의해서 사용할 수 있을까요? 예를 들어 285번 처럼요.**
  - 네 가능합니다. 표준 HTTP 응답 코드 외에 사용자 정의 코드를 추가하여 특정 상황에 대한 응답을 나타낼 수 있습니다. 클라이언트와 서버 모두에서 이해되고 의미를 명확하게 전달 할 수 있는지 고려하면 좋습니다.
